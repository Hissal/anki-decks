# Anki Decks

Personal Chinese-learning Anki decks. Three TSV files, append-only, imported
into Anki on demand.

For the project rationale, scope guardrails, and full schema reference, see
[`CLAUDE.md`](CLAUDE.md). For the canonical tag list, see
[`TAGS.md`](TAGS.md).

## Decks

| File | Purpose |
|------|---------|
| `Chinese_Core_Conversation.tsv` | Everyday useful vocab and spoken phrases. |
| `Chinese_Idioms_Proverbs_Classical.tsv` | Chengyu, proverbs, classical / literary. |
| `Chinese_Slang_Dialect_Flavor.tsv` | Slang, dialect, internet, rude / playful flavor. |

All three share the schema: `Hanzi  Pinyin  English  Note  Audio  Tags` (tabs).

## Adding a card

The TSV files are append-only. Two ways to add a row:

**Interactive helper (recommended):**
```powershell
python scripts/add.py
# or pre-pick the deck:
python scripts/add.py core
python scripts/add.py idioms
python scripts/add.py slang
```
The helper prompts for each field, blocks duplicate Hanzi across all decks,
warns on unknown tags, and confirms before writing.

**By hand:** append a single tab-separated line to the chosen TSV. Make sure
column 6 (Tags) has a tier tag — one of `production-ready`,
`recognition-ready`, `recognition-first` — and only tags that appear in
`TAGS.md`. Then run the validator.

## Validating

Before committing, run:
```powershell
python scripts/validate.py
```
Hard errors: wrong column count, missing required fields, duplicate Hanzi
(within or across decks), header drift. Soft warnings: missing tier tag,
unknown tag, digit-tone pinyin, ASCII letters in Hanzi.

Use `--strict` to make warnings fail too.

## Regenerating the index

`INDEX.md` is a readonly browse view grouped by deck and tag. Source TSV order
is never touched.

```powershell
python scripts/index.py
```

Regenerate after appending new rows. Commit `INDEX.md` alongside the TSV
changes if you want the index to stay current in git.

## Anki import settings

The TSV files are designed so that re-importing **updates** existing notes
rather than creating duplicates. Use these exact settings:

1. **File → Import…** (or drag the TSV onto Anki's main window).
2. **Type:** the same note type each time (e.g. create one called
   `Chinese-Deck` with fields `Hanzi`, `Pinyin`, `English`, `Note`, `Audio`,
   `Tags` in that order).
3. **Deck:** point each TSV at its own deck.
4. **Fields separated by:** Tab.
5. **Allow HTML in fields:** off (unless you've added HTML on purpose).
6. **Field mapping:** column 1 → Hanzi, column 2 → Pinyin, column 3 → English,
   column 4 → Note, column 5 → Audio, column 6 → **Tags** (use the "map to
   Tags" option, not a normal field).
7. **Existing notes:** **Update**. This is the important one — it makes Anki
   match on the first field (Hanzi) and update the existing note instead of
   creating a duplicate.
8. **First field is:** Hanzi — Anki uses this as the match key.

After the first import the settings are remembered per note type. Subsequent
imports are just **File → Import → pick the TSV → Import**.

### Audio

Column 5 (`Audio`) is currently empty everywhere. Audio is generated by
[HyperTTS](https://www.hyper-tts.com/) inside Anki rather than baked into the
TSV, so the column stays unused. If audio ever moves into the TSV, the format
is Anki's `[sound:filename.mp3]` syntax with the file already present in
`collection.media`.

## Git hook

`.githooks/pre-commit` runs the validator on every commit. Enable it once per
clone with:
```powershell
git config core.hooksPath .githooks
```

After that, commits that introduce a hard error are blocked. Warnings are
printed but do not block.

## Scripts reference

| Script | Purpose |
|--------|---------|
| `scripts/add.py` | Interactive row append with dedupe and tag check. |
| `scripts/validate.py` | Validate every TSV. `--strict` makes warnings fatal. |
| `scripts/index.py` | Regenerate `INDEX.md` grouped by deck and tag. |
| `scripts/common.py` | Shared helpers — not run directly. |

All scripts target Python 3.10+ with no external dependencies.
