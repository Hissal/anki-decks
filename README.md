# Anki Decks

Personal Chinese-learning Anki decks. Three TSV files, append-only, imported
into Anki on demand.

For the project rationale, scope guardrails, and full schema reference, see
[`CLAUDE.md`](CLAUDE.md). For the canonical tag list, see
[`TAGS.md`](TAGS.md).

## Decks

| File | Purpose |
|------|---------|
| `Chinese_Core_Conversation.tsv` | Everyday useful vocab and spoken phrases. |
| `Chinese_Idioms_Proverbs_Classical.tsv` | Chengyu, proverbs, classical / literary. |
| `Chinese_Slang_Dialect_Flavor.tsv` | Slang, dialect, internet, rude / playful flavor. |

All three share the schema:
`Hanzi  Pinyin  English  Breakdown  Examples  Note  Link  Tags` (tabs). Each
file starts with an Anki directive block (`#separator:tab`, `#html:true`,
`#columns:…`, `#tags column:8`) that tells Anki how to import without manual
field mapping. The `Audio` and `PersonalNote` fields on the note type have no
TSV column — Anki leaves them untouched on re-import, so existing audio and
personal annotations are preserved.

**Pinyin must be strictly one space-separated syllable per CJK character**
(e.g. `那个` → `nà ge`, not `nèige`). The ruby template aligns each syllable
above each character. The validator enforces this.

## Adding a card

The TSV files are append-only. Two ways to add a row:

**Interactive helper (recommended):**
```powershell
python scripts/add.py
# or pre-pick the deck:
python scripts/add.py core
python scripts/add.py idioms
python scripts/add.py slang
```
The helper prompts for each field, blocks duplicate Hanzi across all decks,
warns on unknown tags, and confirms before writing.

**By hand:** append a single tab-separated line to the chosen TSV. Make sure
column 8 (Tags) has a tier tag — one of `production-ready`,
`recognition-ready`, `recognition-first` — and only tags that appear in
`TAGS.md`. Then run the validator.

## Validating

Before committing, run:
```powershell
python scripts/validate.py
```
Hard errors: wrong column count, missing required fields, duplicate Hanzi
(within or across decks), header drift. Soft warnings: missing tier tag,
unknown tag, digit-tone pinyin, ASCII letters in Hanzi.

Use `--strict` to make warnings fail too.

## Regenerating the index

`INDEX.md` is a readonly browse view grouped by deck and tag. Source TSV order
is never touched.

```powershell
python scripts/index.py
```

Regenerate after appending new rows. Commit `INDEX.md` alongside the TSV
changes if you want the index to stay current in git.

## Anki import settings

The TSV files are designed so that re-importing **updates** existing notes
rather than creating duplicates. The directive block at the top of each file
(`#separator:tab`, `#html:true`, `#columns:Hanzi…Tags`, `#tags column:8`)
tells Anki the separator, column-to-field mapping, and which column is tags —
so the field mapping no longer needs to be set by hand each import.

1. **Note type (one-time setup):** install the `Chinese (anki-decks)` note
   type from [`note_type/Chinese_anki_decks/`](note_type/Chinese_anki_decks/).
   See that directory's README for templates, fields, and the `_ruby.js`
   media-folder file. Field order: `Hanzi`, `Pinyin`, `English`, `Breakdown`,
   `Examples`, `Note`, `Link`, `PersonalNote`, `Audio`. The last two have no
   TSV column — they live only inside Anki.
2. **File → Import…** (or drag the TSV onto Anki's main window).
3. **Deck:** point each TSV at its own deck.
4. **Existing notes:** **Update**. Anki matches on the first field (Hanzi)
   and updates the existing note instead of creating a duplicate.
   `PersonalNote` and `Audio` are preserved across re-imports because no TSV
   column maps to them.

After the first import the settings are remembered per note type. Subsequent
imports are just **File → Import → pick the TSV → Import**.

### Audio

The note type has an `Audio` field, but no corresponding TSV column. Audio is
generated by [HyperTTS](https://www.hyper-tts.com/) inside Anki rather than
baked into the TSV. Because the TSV doesn't include an Audio column, re-import
never overwrites existing audio.

## Git hook

`.githooks/pre-commit` runs the validator on every commit. Enable it once per
clone with:
```powershell
git config core.hooksPath .githooks
```

After that, commits that introduce a hard error are blocked. Warnings are
printed but do not block.

## Scripts reference

| Script | Purpose |
|--------|---------|
| `scripts/add.py` | Interactive row append with dedupe and tag check. |
| `scripts/validate.py` | Validate every TSV. `--strict` makes warnings fatal. |
| `scripts/index.py` | Regenerate `INDEX.md` grouped by deck and tag. |
| `scripts/common.py` | Shared helpers — not run directly. |
| `scripts/migrate_pinyin_per_char.py` | One-shot migration: rewrite the Pinyin column to one space-separated syllable per CJK character. Requires `pip install pypinyin`. |
| `scripts/migrate_note_split.py` | One-shot migration: split the old `Note` column into the new `Breakdown` / `Examples` / `Note` / `Link` columns. |

`add.py`, `validate.py`, `index.py`, and `common.py` target Python 3.10+ with
no external dependencies. The migration scripts additionally need `pypinyin`.
